# M3c Search — Backend FTS Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a public, ranked full-text product search endpoint (`GET /products/search`) backed by a swappable `ProductSearch` provider with a Postgres GIN FTS default implementation.

**Architecture:** A new `search` NestJS module owns a `ProductSearch` injection-token interface (ADR-009), a `PostgresProductSearch` default impl (ADR-011) using raw parameterized SQL for ranked matching, and a `@Public` controller mounting `GET /products/search`. A dedicated `CONCURRENTLY` migration adds the K2 GIN index. The existing `GET /products` catalog list is untouched (no regression).

**Tech Stack:** NestJS 11, TypeScript (strict), Prisma 7 (`@prisma/adapter-pg`), PostgreSQL FTS (`to_tsvector`/`websearch_to_tsquery`/`ts_rank`/GIN), Jest (unit), class-validator/class-transformer (DTO), `curl` (HTTP smoke).

## Global Constraints

- Strict TypeScript; **no `any`** (use `as never` only in test mocks, matching existing specs).
- Prisma 7: `PrismaService extends PrismaClient` (driver adapter already wired). Use `prisma.$queryRaw` (tagged template, auto-parameterized) for raw SQL — never string-interpolate the user query.
- Search is **public** (`@Public()`), **ACTIVE-only** (`status = 'ACTIVE'`), **all sellers**, no auth/seller-scope.
- Response shape must be the catalog `Paginated<Product>` envelope with `PRODUCT_INCLUDE` relations (category, images, seller{displayName,slug}).
- DB is **`ecom_dev`** (shadow `ecom_shadow`), user `sotsys033`, no password. **Never** touch the unrelated `ecomm` DB.
- Verify with `npx tsc --noEmit` (memory: `nest build` exits 0 despite tsc errors) **and** a real boot+HTTP smoke before "done".
- No `git push` without explicit user permission. Commit locally per task.
- Compiled entry is `dist/src/main.js`; smoke-run with `npm run start:dev`.

```typescript
import { Injectable } from '@nestjs/common';
import { Prisma, type Product } from '@prisma/client';
import { PrismaService } from '../prisma/prisma.service';
import { ProductSearch, ProductSearchResult } from './product-search';

/** Relations included so a search hit renders identically to a catalog card. */
const PRODUCT_INCLUDE = {
  category: true,
  images: { orderBy: { position: 'asc' as const } },
  seller: { select: { displayName: true, slug: true } },
} satisfies Prisma.ProductInclude;

/** One ranked, paginated row from the FTS query. */
interface RankedRow {
  id: string;
  rank: number;
  total: bigint;
}

/**
 * Postgres GIN full-text implementation of `ProductSearch` (ADR-011).
 * Two steps: (1) raw parameterized SQL ranks + paginates matching product IDs
 * (the `@@` filter matches the K2 index expression so the GIN index is used);
 * (2) Prisma hydrates the page with relations, re-sorted into rank order.
 * Public, ACTIVE-only, across all sellers. A blank query yields an empty page.
 */
@Injectable()
export class PostgresProductSearch implements ProductSearch {
  constructor(private readonly prisma: PrismaService) {}

  async search(
    q: string,
    page: number,
    pageSize: number,
  ): Promise<ProductSearchResult> {
    const term = (q ?? '').trim();
    if (term === '') {
      return { data: [], page, pageSize, total: 0, totalPages: 1 };
    }

    const offset = (page - 1) * pageSize;

    // $1 = user query (parameterized — no injection). The match expression is
    // identical to the K2 index expression; the rank expression is weighted
    // (name 'A' > description 'B') and computed only on candidate rows.
    const rows = await this.prisma.$queryRaw<RankedRow[]>`
      SELECT p.id,
             ts_rank(
               setweight(to_tsvector('english', p.name), 'A') ||
               setweight(to_tsvector('english', coalesce(p.description, '')), 'B'),
               websearch_to_tsquery('english', ${term})
             ) AS rank,
             count(*) OVER() AS total
      FROM "Product" p
      WHERE p."deletedAt" IS NULL
        AND p.status = 'ACTIVE'
        AND to_tsvector('english', p.name || ' ' || coalesce(p.description, ''))
            @@ websearch_to_tsquery('english', ${term})
      ORDER BY rank DESC, p."createdAt" DESC
      LIMIT ${pageSize} OFFSET ${offset}
    `;

    if (rows.length === 0) {
      return { data: [], page, pageSize, total: 0, totalPages: 1 };
    }

    const total = Number(rows[0].total);
    const ids = rows.map((r) => r.id);

    const products = await this.prisma.product.findMany({
      where: { id: { in: ids } },
      include: PRODUCT_INCLUDE,
    });

    // `IN (...)` does not preserve order — re-sort into the ranked id order.
    const byId = new Map(products.map((p) => [p.id, p]));
    const data = ids
      .map((id) => byId.get(id))
      .filter((p): p is Product => p !== undefined);

    return {
      data,
      page,
      pageSize,
      total,
      totalPages: Math.max(1, Math.ceil(total / pageSize)),
    };
  }
}
```

```typescript
import { Controller, Get, Inject, Query } from '@nestjs/common';
import { Public } from '../auth/decorators/public.decorator';
import { ProductSearch, PRODUCT_SEARCH } from './product-search';
import { SearchProductsDto } from './dto/search-products.dto';

/**
 * Public product search. `GET /products/search` — the static `search` segment
 * is more specific than `ProductsController`'s `@Get(':id')`, so Nest matches
 * it first (verified in the HTTP smoke).
 */
@Controller('products')
export class SearchController {
  constructor(
    @Inject(PRODUCT_SEARCH) private readonly productSearch: ProductSearch,
  ) {}

  @Public()
  @Get('search')
  search(@Query() query: SearchProductsDto) {
    return this.productSearch.search(
      query.q ?? '',
      query.page ?? 1,
      query.pageSize ?? 20,
    );
  }
}
```
